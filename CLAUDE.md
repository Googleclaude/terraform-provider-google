## What this is

The HashiCorp/Google `google` Terraform provider — generally available GCP resources. Beta features live in the sibling `terraform-provider-google-beta` repository. The provider serves as a tf5 mux of two implementations:

- **SDKv2** primary (`google/provider`) — most resources still live here.
- **Plugin Framework** secondary (`google/fwprovider`) — newer resources/data sources.

`main.go` muxes both via `tf5muxserver` and serves at `registry.terraform.io/hashicorp/google`.

### Running specific unit tests

`make test` runs `scripts/get_unit_test_pkgs.sh`, which **excludes acceptance tests by package**: a service package is included only if it has a non-`TestAcc*` test function. Override the package set with `TEST=`:

```sh
make test TEST=./google/services/compute
make testnolint TEST=./google/services/compute TESTARGS='-run TestComputeInstance_basic -v'
```

For accept tests on a single resource:

```sh
make testacc TEST=./google/services/compute TESTARGS='-run TestAccComputeInstance_basic -v'
```

Acceptance tests (`TestAcc*`) hit live GCP. Most require env vars (`GOOGLE_PROJECT`, `GOOGLE_CREDENTIALS`/`GOOGLE_APPLICATION_CREDENTIALS`, `GOOGLE_REGION`, `GOOGLE_ZONE`, `GOOGLE_BILLING_ACCOUNT`, …); see `google/envvar` for the full list and `google/acctest` for fixtures.

## Repository layout

- `main.go` — provider entry point (muxes SDK + framework providers).
- `google/provider/` — SDKv2 provider definition + provider-level tests.
- `google/fwprovider/`, `google/fwresource/`, `google/fwmodels/`, `google/fwtransport/`, `google/fwutils/`, `google/fwvalidators/` — Plugin Framework infrastructure.
- `google/services/<service>/` — one directory per GCP service (~176 services: `compute`, `bigquery`, `storage`, `iam`, …). Each contains resource files, data sources, helpers, and `*_test.go` (unit) / `*_acc_test.go` (acceptance) pairs.
- `google/acctest/` — shared test scaffolding for acceptance tests.
- `google/envvar/` — typed wrappers around the `GOOGLE_*` test env vars.
- `google/sweeper/` — orphaned-resource cleanup used between runs.
- `google/transport/`, `google/verify/`, `google/tpgresource/`, `google/tpgiamresource/`, `google/tpgdclresource/` — shared HTTP transport, validators, and IAM/DCL helpers.
- `google/registry/` — provider/resource registration.
- `scripts/` — `get_unit_test_pkgs.sh`, `gofmtcheck.sh`, `docscheck.sh`, `affectedtests/`, `teamcitytestscripts/`, `diff.go`, `run_diff.sh`.
- `.teamcity/` — TeamCity build configuration.
- `examples/` — example Terraform configurations exercised by docs/tests.

## Changelog entries

Each PR adds a numbered file under `.changelog/` matching the **PR number** (e.g. `.changelog/12345.txt`). Body uses the standard go-changelog block format:

```
` ``release-note:enhancement
container: promote `enable_confidential_storage` from `node_config` in `google_container_cluster` and `google_container_node_pool` to GA (ga)
` ``
```

Allowed types include `release-note:enhancement`, `release-note:new-resource`, `release-note:new-datasource`, `release-note:bug`, `release-note:breaking-change`, `release-note:note`, `release-note:none`. The aggregate `CHANGELOG.md` is regenerated at release; **edit only the per-PR file**, not `CHANGELOG.md` directly. Historical changelogs are split into `CHANGELOG_v0.md` … `CHANGELOG_v5.md`.

## When making a change

1. If the change should also exist in `google-beta`, make it in **magic-modules** (the upstream generator) and let it propagate.
2. If you must hot-patch this repo, edit the corresponding handwritten or generated file, **and** open a PR upstream so the next regeneration doesn't blow it away.
3. Add a changelog entry under `.changelog/<PR#>.txt`.
4. Run `make lint test` (and `make testacc` for the affected resource if you have GCP credentials).
